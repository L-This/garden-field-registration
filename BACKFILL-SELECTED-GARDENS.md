# تعويض التقارير: كامل المواقع أو مواقع مختارة

## ما تم تنفيذه

- نافذة تعويض مرتبطة بمشروع وتاريخ سابق ووقت انتهاء.
- خياران للنطاق:
  - `all`: جميع المواقع المجدولة في ذلك التاريخ حسب جدول الري.
  - `selected`: مواقع تختارها الإدارة يدويًا من مواقع المشروع النشطة.
- صفحة المقاول تعرض فقط المواقع المسموح بها في نافذة التعويض.
- عند الإرسال تتحقق قاعدة البيانات من كل موقع، ولا تقبل أي موقع خارج النطاق.
- العدد المتوقع يُحسب من قاعدة البيانات؛ إذا عدّلت الإدارة النطاق أثناء تجهيز المقاول، يتوقف الإرسال ويطلب تحديث الصفحة.
- النافذة تُغلق تلقائيًا بعد اعتماد التقرير، أو يمكن إغلاقها يدويًا.

## خطوات قاعدة البيانات

نفّذ ملف `supabase-daily-report-logic.sql` كاملًا في SQL Editor. الملف يستخدم `CREATE IF NOT EXISTS` و`ALTER ... IF NOT EXISTS` ولا يحذف التقارير الحالية.

## دمج لوحة الإدارة

تم إرفاق مكوّن جاهز:

`src/components/BackfillAdminPanel.tsx`

انسخه إلى مشروع لوحة الإدارة، ثم استورده في قسم تقارير التعويض:

```tsx
<BackfillAdminPanel openedBy={user?.username || "مدير النظام"} />
```

المكوّن يوفر:

- اختيار المشروع.
- اختيار تاريخ سابق.
- تحديد مدة الفتح.
- فتح كامل المواقع المجدولة.
- أو اختيار حدائق بعينها، مع بحث وتحديد الكل وإلغاء التحديد.
- ملاحظة إدارية.

## إغلاق نافذة يدويًا

```ts
await supabase.rpc("close_daily_report_backfill", {
  p_window_id: windowId,
});
```

## عرض سجل التعويضات

```ts
const { data } = await supabase.rpc("list_daily_report_backfills", {
  p_project_id: null,
});
```

## سلوك صفحة المقاول

عند وجود نافذة تعويض مختارة، يظهر شريط برتقالي يوضح:

- تاريخ التقرير السابق.
- أن النطاق هو مواقع مختارة.
- عدد المواقع المفتوحة.
- الملاحظة الإدارية إن وُجدت.

ولا تظهر له أي حديقة أخرى.
